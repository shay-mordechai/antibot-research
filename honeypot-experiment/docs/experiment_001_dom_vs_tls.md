# 🧪 Experiment 001: DOM/V8 Fingerprinting vs. TLS Impersonation

## 📖 Objective
Modern Web Application Firewalls (WAFs) like Cloudflare do not rely solely on IP reputation. They verify the integrity of the client by interrogating its execution environment.

This experiment compares two vectors:
1. **The Browser Execution Environment (V8/Renderer):** How static DOM properties flag automated headless processes.
2. **The Network Boundary (TLS/JA3):** How perfect cryptographic impersonation can bypass JS Engine execution entirely.

## 🔬 Methodology

### 1. The V8/DOM Test (Playwright)
We launched a headless Chromium instance. In standard browser architecture, the Web page runs within the **Renderer Process** (in an isolated Sandbox). WAFs inject a `cf-challenge` script into the V8 engine, which queries properties like `navigator.webdriver`, hardware concurrency, and WebGL metadata. 
* **Hypothesis:** The WAF will detect the headless execution environment anomalies and drop the connection.

### 2. The Network Impersonation Test (curl_cffi)
Instead of spinning up a full V8 engine and Renderer, we intercept the connection at the **Network Boundary**. We spoofed the TLS Handshake (JA3) and HTTP/2 frames to perfectly mimic a Google Chrome 120 client.
* **Hypothesis:** By perfectly aligning the network fingerprint, the Edge server will grant access (Trust) without needing to interrogate the V8 engine via a JS challenge.

## Research Conclusions

    The Vulnerability of Headless Architectures: Modifying the User-Agent string is entirely insufficient. WAFs evaluate the integrity of the Renderer process itself. If the OS graphics stack (WebGL) or V8 properties (Webdriver flag) don't perfectly align with the expected OS, the session is blocked.

    Network Layer Supremacy: Bypassing Anti-Bot systems does not necessarily require building a "perfectly undetectable browser." By successfully impersonating the cryptographic TLS handshake (JA3/JA4) and HTTP/2 framing, we proved that WAFs heavily prioritize the Network layer. If the TLS signature is perfectly legitimate, the WAF is significantly less likely to force an aggressive V8/DOM JavaScript challenge.
